# 条形码扫码功能使用说明

## 一、功能概述

条形码扫码功能可以帮助您快速识别商品，完成采购入库和销售出库操作。

**适用场景：**
- 采购入库时快速选择商品
- 销售出库时快速选择商品
- 商品管理时快速录入条形码

**支持的条码格式：**
- Code 128（常用）
- EAN-13（商品条形码）
- EAN-8
- Code 39
- UPC-A

## 二、扫码功能入口

### PC端
1. **采购入库页面**
   - 页面顶部「📷 扫码入库」大按钮
   - 商品列表顶部「📷 扫码」小按钮

2. **销售出库页面**
   - 页面顶部「📷 扫码出库」大按钮
   - 商品列表顶部「📷 扫码」小按钮

3. **商品管理页面**
   - 新增/编辑商品时，商品条形码输入框右侧「📷 扫码」按钮

### 手机端（H5/微信）
入口位置与PC端一致，扫码界面针对移动端优化。

## 三、扫码使用流程

### 采购入库扫码

```
1. 点击「📷 扫码入库」按钮
2. 弹出扫码界面
3. 点击「开始扫码」按钮（iOS必须手动点击）
4. 将商品条形码对准扫描框
5. 识别成功后自动关闭界面
6. 商品自动添加到入库单
7. 填写数量并提交
```

### 销售出库扫码

```
1. 选择客户
2. 点击「📷 扫码出库」按钮
3. 弹出扫码界面
4. 点击「开始扫码」按钮
5. 扫描商品条形码
6. 识别成功后：
   - 商品已建档且有库存 → 自动添加到出库单
   - 商品已建档但无库存 → 提示"库存为0，无法出库"
   - 商品未建档 → 提示"该商品未建档，请先创建商品"
```

### 商品管理扫码

```
1. 进入商品管理
2. 点击「新增商品」或「编辑」
3. 点击条形码输入框右侧「📷 扫码」
4. 扫描商品条形码
5. 条形码自动填充到输入框
6. 填写其他信息并保存
```

## 四、iPhone + 微信特别说明

### 首次使用
1. 首次扫码会弹出摄像头权限请求
2. 点击「允许」授权
3. 如果拒绝：提示「请在系统设置中开启摄像头权限」

### 使用技巧
- **光线要求**：确保环境光线充足
- **距离要求**：条形码距离摄像头15-30厘米
- **角度要求**：保持手机平稳，不要快速移动
- **识别失败**：尝试调整角度或距离，或使用「手动输入」

### 常见问题

| 问题 | 解决方法 |
|------|----------|
| 无法打开摄像头 | 检查浏览器权限设置 |
| 识别不灵敏 | 确保光线充足，条形码完整清晰 |
| 提示权限被拒绝 | 到手机设置 → 微信 → 摄像头 → 允许 |
| 摄像头黑屏 | 刷新页面重试 |

## 五、手动输入备选方案

如果扫码失败，可使用手动输入：
1. 点击扫码界面底部的「手动输入」
2. 输入商品条形码号码
3. 点击「查询」按钮
4. 系统会查找对应的商品

## 六、商品条形码管理

### 新增商品时录入条形码
```
商品管理 → 新增商品 → 填写商品信息 → 扫码或输入条形码 → 保存
```

### 编辑商品时修改条形码
```
商品管理 → 找到商品 → 点击编辑 → 修改条形码 → 保存
```

### 条形码唯一性
- 同一条形码只能绑定一个商品
- 保存时系统会自动检查重复
- 如果重复会提示「该条形码已被其他商品使用」

### 按条形码搜索商品
在商品列表的搜索框中，可以输入条形码进行搜索。

## 七、技术配置

### 扫码组件参数
```javascript
{
  fps: 10,                      // iPhone建议10fps
  qrbox: { width: 280, height: 120 },  // 横向长条适合条形码
  aspectRatio: 1.777,           // 16:9比例
  formats: ['code_128', 'ean_13', 'ean_8', 'code_39', 'upc_a'],
  facingMode: 'environment'      // 后置摄像头
}
```

### 依赖安装
如果缺少 html5-qrcode 依赖，执行：
```bash
npm install html5-qrcode
```

## 八、数据库变更

需要在 Supabase 数据库中执行以下 SQL 添加 barcode 字段：

```sql
-- 为 products 表添加 barcode 字段
ALTER TABLE products ADD COLUMN IF NOT EXISTS barcode VARCHAR(50);

-- 创建索引
CREATE INDEX IF NOT EXISTS idx_products_barcode ON products(barcode) WHERE barcode IS NOT NULL;

-- 创建唯一索引（确保条形码唯一）
CREATE UNIQUE INDEX IF NOT EXISTS idx_products_barcode_unique 
ON products(barcode) 
WHERE barcode IS NOT NULL AND barcode != '';
```

## 九、使用小贴士

1. **提前录入条形码**：在商品管理中提前为商品绑定条形码
2. **打印标签**：为商品打印条形码标签，方便扫描
3. **批量操作**：扫码适合单件操作，批量入库建议使用列表选择
4. **网络要求**：扫码功能需要网络连接查询商品信息

## 十、故障排除

| 症状 | 可能原因 | 解决方案 |
|------|----------|----------|
| 扫码界面打不开 | 浏览器兼容问题 | 尝试刷新页面或更换浏览器 |
| 摄像头一直黑屏 | 权限未授权 | 检查系统设置允许摄像头 |
| 识别成功率低 | 光线不足/条形码损坏 | 改善光线或手动输入 |
| 商品找不到 | 商品未建档或条形码未绑定 | 先到商品管理创建/绑定 |
| 震动没反应 | 手机不支持震动功能 | 正常现象，不影响使用 |
